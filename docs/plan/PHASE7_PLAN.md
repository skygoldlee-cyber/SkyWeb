# Phase 7 실행계획 — 규칙 기반 추천 + 최소 관리자

> **상위 문서**: PLAN.md §5(recommendations), §7(기능 4·6), §2(SQLAdmin)
> **기간(현실안)**: 1–2주
> **완료 기준**: 점수 구간 기반 추천 노출

운영을 위한 최소한의 관리자 조회 화면(SQLAdmin)을 붙인다.

## 2. 작업 단위

### 2.0 의존성 설치
```bash
pip install sqladmin  # 관리자 패널
```

### 2.1 recommendations 스키마 (PLAN.md §5)
- [ ] `recommendations` ORM: `id`, `analysis_id FK`, `product_name`, `category`, `reason`, `created_at`
- [ ] Alembic 마이그레이션

### 2.2 규칙 기반 추천 엔진 (v1)
- [ ] 규칙 정의 방식 결정: 코드 상수 vs 설정 파일(YAML/JSON) — 수정 용이성 위해 설정 권장
- [ ] 매핑 로직: 카테고리/항목 점수 구간 → 추천 항목(SSOT로 단일 진입점, 과거 `create_prescription()` 패턴 준용)
- [ ] 추천 생성 시점 결정: 분석 완료 시 워커가 함께 저장 vs 조회 시 계산
      (재현성·일관성 위해 **저장** 권장)
- [ ] 빈/경계 케이스: 추천 없음, 동률 구간 처리

### 2.3 추천 API & 노출
- [ ] 단건 결과 응답에 추천 포함 또는 `GET /api/v1/analyses/{id}` 확장
- [ ] 프론트: 결과 화면(Phase 5)에 추천 카드 섹션(제품명·카테고리·사유)
- [ ] **표현 규제(§8)**: "치료" 아닌 "관리/추천", 효능 단정 금지

### 2.4 최소 관리자 (SQLAdmin)
- [ ] SQLAdmin 연결(users/analyses/recommendations 조회)
- [ ] **접근 통제**: 관리자 인증 필수(공개 금지), 운영에서만 노출
  ```bash
  # .env
  ADMIN_USERNAME=admin
  ADMIN_PASSWORD=change-me-in-production
  ADMIN_SECRET_KEY=your-admin-secret-key
  ```
- [ ] 간단 통계(분석 건수/실패율 등)는 조회 위주로
- [ ] 민감정보(얼굴 이미지) 직접 노출 최소화(서명 URL·권한 확인)

### 2.5 PWA 푸시 알림 (Phase 4: 푸시 알림)
- [ ] Notification API 권한 요청 구현
- [ ] 백엔드 푸시 서비스 연동 준비 (VAPID 키 설정)
- [ ] 분석 완료 시 푸시 알림 트리거
- [ ] 추천 업데이트 시 푸시 알림 트리거
- [ ] 알림 클릭 시 딥링크 처리 (결과 페이지로 이동)
- [ ] 푸시 알림 권한 관리 UI

## 4. 완료 기준 (Definition of Done)

- [ ] 분석 결과의 점수 구간에 따라 추천이 생성·저장됨
- [ ] 결과 화면에 추천 카드가 노출됨
- [ ] 관리자 화면에서 users/analyses/recommendations 조회 가능(인증 필요)
- [ ] 추천 문구에 "진단/치료" 단정 표현 없음
- [ ] 푸시 알림 권한 요청이 정상 작동
- [ ] 분석 완료/추천 업데이트 시 푸시 알림 수신

### 테스트 전략
- 규칙 기반 추천 엔진 테스트 (점수 구간 → 추천 매핑)
- 추천 SSOT 테스트 (단일 진입점 확인)
- SQLAdmin 인증 가드 테스트 (미인증 접근 차단)
- 추천 문구 규제 준수 테스트
- 푸시 알림 권한 요청 테스트
- 푸시 알림 수신 테스트

## 5. 리스크 / 주의

- **과설계 금지**: v1은 규칙 추천. ML/개인화·A-B 테스트는 부록 C(v2).
- **규칙 SSOT**: 추천 매핑이 여러 곳에 흩어지면 유지보수 지옥(과거 SSOT 위반 사례) → 단일 소스.
- **관리자 보안**: SQLAdmin은 강력하므로 인증·권한 없이 절대 노출 금지(민감정보 다룸).
- **법적 표현**: 화장품/효능 광고 표현 규제 — "추정·관리" 톤 유지.
