# Phase 5 실행계획 — 결과 시각화 (리포트 / 히트맵 / 그래프)

> **상위 문서**: PLAN.md §3(흐름 4), §7(핵심 기능 3), Phase 3 저장 구조
> **기간(현실안)**: 2주
> **완료 기준**: 단건 결과 화면 완성
> **성격**: 워커가 저장한 점수/히트맵을 사람이 이해할 화면으로 렌더. 비동기 완료 감지 포함.

---

## 0. 목표 한 줄

`analysis_id` 한 건의 분석 결과(10 카테고리 + 18 리포트 항목 + 히트맵)를 보기 좋게 시각화하고,
분석 진행 상태를 폴링/SSE로 추적해 완료 시 자동 표시한다.

## 2. 작업 단위

```bash
npm install recharts  # 차트 라이브러리
```

### 2.1 상태 추적 (비동기 완료 감지) — PLAN.md §3-4

1. `/analyses/[id]` 단건 결과 화면(상태추적 → 리포트 → 히트맵)

4. 결과 Zod 스키마

---

## 4. 완료 기준 (Definition of Done)

- [ ] 업로드 직후 진입 시 `processing` 상태가 표시되고, 완료되면 자동으로 결과로 전환
- [ ] 10 카테고리·18 항목·`overall_score`가 정확히 렌더(백엔드 값과 일치)
- [ ] 히트맵이 서명 URL로 안전하게 표시(직접 공개 URL 아님)
- [ ] `failed` 케이스가 사용자 친화적으로 안내됨
- [ ] 화면 어디에도 "진단" 표현 없음

### 테스트 전략
- 폴링 동작 테스트 (processing → done 전이)
- Zod 스키마 검증 테스트 (JSONB 키 불일치 감지)
- 히트맵 서명 URL 만료 테스트
- 반응형 레이아웃 테스트 (모바일/데스크톱)

---

## 5. 리스크 / 주의

- **폴링 과다**: 간격·중단 조건(완료/실패/이탈) 관리. 무한 폴링 방지.
- **JSONB 키 불일치**: Phase 3 저장 키와 프론트 기대 키가 어긋나면 화면이 빈다 → 스키마로 강제 검증.
- **히트맵 URL 만료**: 서명 URL 만료 시간과 사용자가 화면에 머무는 시간 정합.
- **소수 정밀도**: `overall_score`는 NUMERIC → 반올림 표기는 표시 단계에서만, 원값 보존.
